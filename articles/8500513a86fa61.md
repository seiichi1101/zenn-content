## Java

- Stream API (Java 8 以降)
- ReferencePipeline と Sink インターフェースを使用して遅延評価を実現
- 中間操作と終端操作の概念

## Kotlin

- `Sequence` が Java Stream に相当
- `sequence` ビルダーを使って lazy evaluation を実現
- Java との相互運用性が高い

## Python

- Generator 関数 (`yield` キーワード)
- より簡潔な構文で同様の遅延評価が可能

## JavaScript

- Iterator プロトコル
- Node.js 20 以降では Async Iterator もサポート
- Node.js 22 では `ReadableStream.values()` メソッドにより配列操作が簡潔に

これらの言語では、構文レベルでの遅延評価サポートがあるため、Java のような複雑なパイプライン機構は不要です。
しかし、内部的な概念(データの段階的処理、遅延評価)は共通しています。
